1 Linked List hota kya hai?

Array mein socho lockers ek line mein fix hote hain. Linked List mein socho ek treasure hunt — har clue (node) tumhe agla clue kahan milega woh batata hai. Tumhe pura raasta pata nahi hota, bas itna pata hota hai "agla kahan hai".

Har node do cheezein store karta hai: ek value (data), aur ek pointer jo agle node ka address batata hai. Last node ka pointer null hota hai — matlab "yahan list khatam".

Java mein ek node aisa dikhta hai:

class ListNode { int val; ListNode next; ListNode(int val) { this.val = val; this.next = null; } }

List access karne ke liye sirf head (pehla node) ka reference chahiye hota hai — wahan se .next follow karke poori list traverse kar sakte ho.

2 Array vs Linked List — Kab Kya Use Karein
OperationArrayLinked List
Access by indexO(1) — fastO(n) — slow, traverse karna padega
Insert/Delete at startO(n) — sab shift karna padegaO(1) — bas pointer change
Insert/Delete at endO(1)O(n) (without tail pointer)
MemoryContinuous blockScattered, extra pointer memory
SizeFixed (or costly resize)Dynamic, easy grow/shrink

Simple rule: agar baar-baar beech mein insert/delete karna ho, Linked List behtar hai. Agar baar-baar random index access karna ho, Array behtar hai.

3 Reverse Karna — 3 Pointer Technique
Yeh sबसे important pattern hai
Linked List reverse karna matlab har node ka next pointer ulta kar dena — jo node pehle aage point kar raha tha, ab woh peeche point kare. Iske liye 3 pointers chahiye: prev, curr, next.

Example: 1 → 2 → 3 → null ko 3 → 2 → 1 → null banana hai।

StepprevcurrAction
Startnull1curr.next ko prev (null) banao
1121 ab kisi ko point nahi karta (list ka end ban gaya), curr.next ko prev (1) banao
2232.next ab 1 hai, curr.next ko prev (2) banao
End3null3.next ab 2 hai — prev (3) hi naya head hai!

Logic: har step pe, current node ka next "prev" ki taraf mod do, fir dono pointers ek-ek step aage badha do.

ListNode prev = null, curr = head; while (curr != null) { ListNode next = curr.next; // agle node ko bhool jaane se pehle save karo curr.next = prev; // pointer ulta karo prev = curr; // prev ko aage badhao curr = next; // curr ko aage badhao } return prev; // prev hi naya head hai
⚠️ Sabse common mistakecurr.next ko ulta karne se pehle agle node (next) ko save karna mat bhoolo — warna ek baar pointer ulta karne ke baad tum list ka baaki hissa "kho" dete ho, kyunki curr.next ab kisi aur ko point kar raha hai.

4 Fast-Slow Pointer (Floyd's Algorithm)
Simple shabdon mein
Do pointers ek saath chalte hain — ek "slow" (1 step at a time), ek "fast" (2 steps at a time). Socho race track pe ek runner dusre se double speed se daud raha hai — agar track circular hai (cycle hai), fast wala slow ko ek na ek din pakad lega.

Agar list mein cycle hai (koi node wapas peeche kisi node ki taraf point kar raha hai), fast pointer kabhi null tak nahi pahunchega — balki ek din slow pointer ko "pakad" lega, kyunki woh usse dogni speed se chal raha hai aur cycle ke andar fasa hua hai.

ListNode slow = head, fast = head; while (fast != null && fast.next != null) { slow = slow.next; // 1 step fast = fast.next.next; // 2 steps if (slow == fast) return true; // mil gaye = cycle hai! } return false; // fast null tak pahunch gaya = cycle nahi hai
💡 Yeh technique aur kahan kaam aati haiFast-slow pointer sirf cycle detection ke liye nahi — list ka middle node dhundne ke liye bhi use hota hai. Jab fast pointer end tak pahunche, slow pointer exactly middle mein hoga (kyunki woh aadhi speed se chal raha tha).

5 Dummy Node Trick
Yeh problem solve karta hai
Jab tumhe list ke head ko hi modify/delete karna ho (jaise pehla node hata dena), code mein "agar yeh head hai toh alag handle karo" jaisa special case baar-baar likhna padta — bahut messy ho jaata hai. Dummy node ye problem khatam kar deta hai.

Trick: ek fake "dummy" node banao jiska next head ki taraf point kare. Ab tumhara "real" starting point dummy hai, head nahi — isliye head ko delete/change karna bhi ek normal case ban jaata hai, special case nahi.

ListNode dummy = new ListNode(0); dummy.next = head; ListNode curr = dummy; // ab curr.next se shuru karke kaam karo // list ke end mein, dummy.next hi tumhara naya head hai return dummy.next;

6 Do Lists ko Merge Karna

Do sorted linked lists ko merge karke ek sorted list banana — bahut common pattern hai (Merge Sort ka base bhi yehi hai). Idea: dummy node se shuru karo, dono lists ke heads compare karo, jo chhota ho usse result mein jodo, us list ko aage badhao, repeat karo.

ListNode dummy = new ListNode(0), curr = dummy; while (a != null && b != null) { if (a.val <= b.val) { curr.next = a; a = a.next; } else { curr.next = b; b = b.next; } curr = curr.next; } curr.next = (a != null) ? a : b; // jo bachi list hai usse attach kar do return dummy.next;

7 Common Mistakes

1. NullPointerException — hamesha check karo node ya node.next null toh nahi, especially fast.next.next jaisi cheez likhte waqt.

2. Pointer ulta karne se pehle next save na karna — Reverse karte waqt sबसे common bug, jisse list ka baaki hissa "lost" ho jaata hai.

3. Head modify karte waqt dummy node use na karna — special-case code likhna padta hai jo error-prone hai.

4. Cycle wali list mein normal traversal try karna — agar list mein cycle hai aur tum normal loop likho, infinite loop ban jaayega.
