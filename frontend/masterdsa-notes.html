📦 1. Array Basics

Array ek collection hota hai jisme same-type elements ek continuous (lagataar) memory block mein store hote hain. Har element ka apna index hota hai, jo 0 se shuru hota hai. Yeh "continuous memory" wali property hi array ko special banati hai — isi wajah se tum kisi bhi index pe O(1) (instant) access kar sakte ho.

Memory mein yeh values ek doosre ke bilkul saath rakhi hoti hain. Agar array ka starting address pata hai, toh arr[i] ka address seedha calculate ho jata hai (base_address + i × size_of_type) — isliye access O(1) hai, chahe array ki size 10 ho ya 10 lakh.

💡 Java/C++ mein array ka size fixed hota hai jab tum banate ho. Python ki list aur Java ka ArrayList dynamically grow karte hain — andar se woh bhi resize karte waqt naya bada array bana ke purana copy karte hain.
👈👉 2. Two Pointer Technique

Jab array sorted ho ya tumhe kisi pair/triplet ko efficiently dhundna ho, do pointers — ek shuru se (left), ek end se (right) — use karke tum O(n²) brute-force ko O(n) mein convert kar sakte ho.

Kaise kaam karta hai — example ke saath

Maan lo tumhe sorted array [2, 7, 11, 15] mein woh do numbers chahiye jinka sum 9 ho.

Step 1: L=2, R=15, sum = 17 — bahut zyada hai, toh R ko ek step left lao.

Step 2: L=2, R=11, sum = 13 — abhi bhi zyada, R ko aur left lao.

Step 3: L=2, R=7, sum = 9 ✅ — answer mil gaya! Bina kisi nested loop ke, sirf 3 steps mein.

int l = 0, r = arr.length - 1; while (l < r) { int sum = arr[l] + arr[r]; if (sum == target) { // found! return l, r } else if (sum < target) { l++; // sum chhota hai, left ko aage badhao (bada number lao) } else { r--; // sum bada hai, right ko peeche lao (chhota number lao) } }
💡 Kaise pehchaano: agar question mein "sorted array" ka zikr ho aur tumhe pair/triplet ka sum, ya array ko reverse karna ho, ya palindrome check karna ho — Two Pointer try karo.

🪟 3. Sliding Window

Yeh technique tab use hoti hai jab tumhe ek contiguous (lagataar) subarray ya substring ka best answer chahiye ho — jaise max sum, min length, ya longest unique substring. "Window" ek range hai jo expand (right pointer aage) aur shrink (left pointer aage) hoti rehti hai.

Example — Minimum Size Subarray with sum >= 7

Array: [2, 3, 1, 2, 4, 3], target sum = 7

StepWindowSumAction
1[2]2Expand right
2[2,3]5Expand right
3[2,3,1]6Expand right
4[2,3,1,2]8Sum>=7! Shrink left → length 4
5[3,1,2]6Sum<7, expand right again
6[3,1,2,4]10Sum>=7! Shrink → [1,2,4]=7, shrink → [2,4]=6 stop. Best length so far = 2

Final answer: length 2 (window [4,3]). Notice — hum kabhi peeche jaake dobara nahi check karte, isliye yeh O(n) hai, na ki O(n²).

int l = 0, sum = 0, minLen = Integer.MAX_VALUE; for (int r = 0; r < arr.length; r++) { sum += arr[r]; // window expand while (sum >= target) { // condition match? shrink karo minLen = Math.min(minLen, r - l + 1); sum -= arr[l]; l++; } }
💡 Pehchaan: agar question mein "contiguous subarray/substring" ho aur kuch maximize/minimize karna ho (length, sum, count of distinct chars) — Sliding Window 90% chance hai ki sahi approach hai.
⚠️ Common mistake: log "while" ki jagah "if" likh dete hain shrink karte waqt — isse window sahi se shrink nahi hota jab multiple shrinks chahiye ho. Hamesha while use karo jab tak condition true ho.

🗂️ 4. HashMap Pattern

Bahut sare array problems mein tumhe yeh yaad rakhna padta hai "kya yeh value pehle dekhi hai?" — HashMap/HashSet isko O(1) mein answer de deta hai (average case), jisse nested loop ki zarurat khatam ho jaati hai.

Classic Example — Two Sum

nums = [2,7,11,15], target = 9. Brute force har pair check karega (O(n²)). HashMap approach: har number ke liye check karo "kya complement (target - num) pehle dekha hai?"

inums[i]complementseen map?Action
027emptyNot found → store {2:0}
172{2:0}Found! 2 is at index 0 → answer [0,1]
Map seen = new HashMap<>(); for (int i = 0; i < nums.length; i++) { int complement = target - nums[i]; if (seen.containsKey(complement)) { return new int[]{seen.get(complement), i}; // found! } seen.put(nums[i], i); // store AFTER checking }
⚠️ Common mistake: complement check karne se pehle current number ko map mein daal dena — isse galat self-match ho sakta hai (jaise target=4, nums=[2] toh 2+2 match kar jayega jo wrong hai). Hamesha check karo, phir store karo.

📈 5. Kadane's Algorithm

Jab tumhe maximum sum wala contiguous subarray dhundna ho, Kadane's Algorithm O(n) mein solve karta hai. Idea: har position pe decide karo — "current element ko pichle subarray mein jodu, ya yahan se naya subarray shuru karu?"

Example: [-2, 1, -3, 4, -1, 2, 1, -5, 4]
iarr[i]curSummaxSumLogic
0-2-2-2Start
1111max(1, -2+1=-1) = 1 → restart better
2-3-21max(-3, 1-3=-2) = -2
3444max(4, -2+4=2) = 4 → restart better
4-1344-1=3, better than starting fresh (-1)
52553+2=5
61665+1=6, new max!

Answer = 6 (subarray [4,-1,2,1]). Notice — jab bhi current sum negative hota, hum usse "discard" kar dete (naya start), kyunki negative sum kabhi future mein madad nahi karega.

int maxSum = arr[0], curSum = arr[0]; for (int i = 1; i < arr.length; i++) { curSum = Math.max(arr[i], curSum + arr[i]); // restart ya continue? maxSum = Math.max(maxSum, curSum); }

➕ 6. Prefix Sum

Agar tumhe baar-baar "index i se j tak ka sum kya hai" jaisa sawaal poocha jaaye, ya "kitne subarrays ka sum exactly X hai" — prefix sum array bana lo jisme prefix[i] = arr[0]+arr[1]+...+arr[i]. Fir koi bhi range sum O(1) mein nikal sakte ho: sum(i,j) = prefix[j] - prefix[i-1].

Combine karke HashMap ke saath, "subarray count with sum = K" jaisa problem bhi O(n) mein solve hota hai — jab bhi (currentSum - K) pehle dekha gaya ho, matlab beech ka subarray sum K hai.

🎯 7. Binary Search on Arrays

Sorted array mein linear search O(n) lagta hai, par binary search O(log n) mein kaam karta hai — har step pe search space half ho jaata hai. mid = (low+high)/2, agar target chhota hai toh right half discard, bada hai toh left half discard.

Rotated sorted array mein thoda twist hai — array poora sorted nahi lagega, par ek half hamesha sorted hoga. Pehle check karo kaunsa half sorted hai, fir decide karo target us half mein hai ya nahi.

Kabhi-kabhi array ko pehle sort karna problem ko bahut simple bana deta hai — jaise 3Sum mein sort karne ke baad two-pointer use kar sakte ho, ya merge intervals mein sort karne ke baad overlapping intervals adjacent ho jaate hain.

💡 Trade-off yaad rakho: sorting O(n log n) lagta hai, toh agar tumhe O(n) solution chahiye (jaise Kadane's), sorting use nahi karna.

⚠️ 9. Common Mistakes (Sabse zyada hone wali galtiyan)

1. Off-by-one errors: while(l < r) vs while(l <= r) — galat condition se infinite loop ya wrong answer aata hai. Hamesha socho: kya l aur r same index pe milne chahiye ya nahi?

2. Empty array check bhool jana: Agar array empty ho toh code crash ho sakta hai. Hamesha edge case socho.

3. Integer overflow: Bade sums ke saath int overflow ho sakta hai — agar values bahut badi hain, long use karo.

4. HashMap mein check aur insert ka order: Jaisa Two Sum mein dekha — pehle check karo, phir insert karo, warna galat self-match ho sakta hai.

5. Sliding window mein "if" vs "while": Shrink karte waqt while use karo, if nahi — taaki multiple shrinks ho sake jab zarurat ho.
