Arrays & Strings

Yeh tumhara DSA ka pehla aur sabse zaroori topic hai. Yahan se shuru karenge bilkul basics se — jaise koi pehli baar padh raha ho. Har pattern ke saath ek real example dekhenge step-by-step, taaki sirf "yaad" na karo, balki samjho ki woh kaam kaise karta hai aur kab use karna hai.

1 Array hota kya hai?

Socho tumhare paas 5 lockers hain, sab ek line mein lage hue, side by side. Har locker ka apna number hai — Locker 0, Locker 1, Locker 2, aur aage. Tum kisi bhi locker ka number bolke seedha us locker tak pohonch sakte ho, baaki lockers khole bina.

Array bilkul yehi hai. Ek array values ka ek group hota hai jo computer ki memory mein ek line mein, side-by-side store hota hai. Har value ka ek index (number) hota hai jo 0 se shuru hota hai — pehla element index 0 pe, doosra index 1 pe, aise hi aage.

010
120
230
340
450

Yahan arr[0] ka matlab hai "index 0 pe jo value hai woh do" — yani 10. Aur arr[3] ka matlab 40. Bas itna simple hai.

Ab sawaal — yeh special kyun hai? Kyunki memory mein yeh values bilkul saath-saath rakhi hoti hain (locker wali line jaisi), computer ko pata hota hai "array index 0 pe yahan se shuru hota hai", aur kisi bhi index ka address woh seedha calculate kar leta hai — kisi cheez ko khojna nahi padta. Isliye arr[3] nikalna utna hi fast hai jitna arr[0] nikalna, chahe array mein 5 elements ho ya 5 crore. Isi ko hum O(1) time kehte hain — matlab "constant time", size se farak nahi padta.

Toh phir array mein slow kya hai?

Agar tumhe beech mein kahin se ek element insert ya delete karna ho (jaise index 2 pe naya element daalna), toh uske baad ke saare elements ko ek-ek position shift karna padega — taaki line mein gap na aaye. Yeh O(n) time leta hai, matlab array jitna bada hoga utna time lagega.

💡 Yaad rakhne wali baatJava aur C++ mein array ki size fix hoti hai jab tum usse banate ho. Python ki list aur Java ka ArrayList dikhne mein "dynamic" lagte hain (size badh sakti hai), par andar se woh bhi jab size badhani hoti hai, ek naya bada array banate hain aur purana data copy kar dete hain.
2 Two Pointer Technique
Simple shabdon mein
Two Pointer ka matlab hai — array mein do jagah par ek-saath "nazar" (pointer) rakhna, jaise ek finger array ke shuru pe aur doosra finger end pe. Dono fingers ko smartly move karke tum bahut sare combinations ko bina check kiye skip kar sakte ho.

Yeh kab use hota hai? Jab array sorted ho aur tumhe kisi pair (ya triplet) ko dhundna ho jiska sum kisi target ke barabar ho — bina har possible pair ko check kiye (jo O(n²) leta, matlab bahut slow).

Chalo example se samajhte hain. Sorted array hai [2, 7, 11, 15], aur humein woh do numbers chahiye jinka sum 9 ho.

Hum do pointers rakhte hain — L (left) shuru mein, R (right) end mein.

L2
7
11
R15

Step 1: arr[L] + arr[R] = 2 + 15 = 17. Yeh target (9) se zyada hai. Matlab sum ko chhota karna padega — toh hum R ko ek step left la dete hain (chhota number layenge).

L2
7
R11
15

Step 2: 2 + 11 = 13. Abhi bhi zyada hai. R ko aur left lao.

L2
R7
11
15

Step 3: 2 + 7 = 9 ✅ Mil gaya! Sirf 3 steps mein answer nikal gaya, bina kisi nested loop ke.

Logic samjho: jab sum target se zyada ho, toh R ko left lao (chhota number layega, sum kam hoga). Jab sum target se kam ho, toh L ko right lao (bada number layega, sum badhega). Array sorted hai isliye yeh trick kaam karti hai.

int l = 0, r = arr.length - 1; while (l < r) { int sum = arr[l] + arr[r]; if (sum == target) { // mil gaya — return karo } else if (sum < target) { l++; // sum chhota hai, bada number chahiye → left aage badhao } else { r--; // sum bada hai, chhota number chahiye → right peeche lao } }
💡 Kaise pehchaano kab use karna haiAgar question mein "sorted array" ka zikr ho, ya tumhe pair/triplet ka sum dhundna ho, array reverse karna ho, ya palindrome check karna ho — Two Pointer ek baar zaroor try karo.

3 Sliding Window
Simple shabdon mein
Ek "window" socho jaise ek rubber band jo array ke kuch consecutive elements ko ghere hue hai. Yeh window kabhi badi hoti hai (expand), kabhi chhoti (shrink), par hamesha continuous elements ko cover karti hai — beech mein koi gap nahi.

Yeh use hota hai jab tumhe ek contiguous (lagataar, beech mein gap na ho) subarray ya substring ka best answer chahiye ho — jaise sabse chhota subarray jiska sum kam se kam X ho, ya sabse lamba substring jisme koi repeat character na ho.

Chalo dekhte hain example se. Array hai [2, 3, 1, 2, 4, 3], aur humein chahiye sabse chhota subarray jiska sum kam se kam 7 ho.

StepWindowSumKya hua
1[2]2Sum kam hai, right ko aage badhao (expand)
2[2,3]5Abhi bhi kam, expand karo
3[2,3,1]6Abhi bhi kam, expand karo
4[2,3,1,2]8Sum>=7 ho gaya! Ab left ko aage badhao (shrink) — length 4 record karo
5[3,1,2]6Sum<7 phir se, expand karo
6[3,1,2,4]10Sum>=7! Shrink karte raho jab tak condition ho: [1,2,4]=7 ✓, fir [2,4]=6 ✗ stop. Best length ab tak 2

Final answer: length 2 (window [4,3], jo array mein aakhri 2 elements hain). Notice karo — hum kabhi peeche jaake purana kaam dobara nahi karte, sirf left aur right pointer hi move karte hain. Isi liye yeh O(n) hai, na ki O(n²).

int l = 0, sum = 0, minLen = Integer.MAX_VALUE; for (int r = 0; r < arr.length; r++) { sum += arr[r]; // window expand — right wala element add karo while (sum >= target) { // condition match? ab shrink karo minLen = Math.min(minLen, r - l + 1); sum -= arr[l]; // left wala element hatao l++; } }
💡 Kaise pehchaano kab use karna haiAgar question mein "contiguous subarray" ya "substring" ka zikr ho, aur tumhe length/sum/count maximize ya minimize karna ho — Sliding Window lagbhag hamesha sahi approach hoti hai.
⚠️ Sabse common mistakeShrink karte waqt log if use kar lete hain while ki jagah. Isse problem yeh hoti hai ki agar ek hi step mein multiple baar shrink karna ho, toh sirf ek baar hoga aur answer galat aa jayega. Hamesha while use karo jab tak condition true rahe.

4 HashMap Pattern
HashMap kya hota hai (basics)
HashMap ek aisi data structure hai jo key → value pairs store karta hai, aur kisi bhi key ki value ko dhundne mein sirf O(1) time leta hai (average case) — chahe usme 10 items ho ya 10 lakh. Socho ek phone ki contact list — naam (key) daalo, number (value) instantly mil jaata hai, poori list scroll nahi karni padti.

Array problems mein bahut baar tumhe yeh yaad rakhna padta hai ki "kya yeh value mainne pehle dekhi hai?" ya "kis index pe dekhi thi?". Normally yeh check karne ke liye tumhe poora array dobara loop karna padega (O(n)), lekin agar tum HashMap mein values store karte jaao jaise-jaise dekhte ho, toh "dekha hai ya nahi" check karna O(1) ho jaata hai.

Yeh nested loops (O(n²)) ko single loop (O(n)) mein convert karne ka sabse common trick hai DSA mein.

Example — Two Sum

Array: nums = [2, 7, 11, 15], target = 9. Humein woh do numbers chahiye jinka sum 9 ho, aur unke indices chahiye.

Brute force tareeka har pair check karega (har i ke liye har j check karo) — O(n²). HashMap tareeka: har number ke liye sochenge "iska partner kya hona chahiye?" (complement = target - num), aur check karenge "kya yeh partner pehle dekha hai?"

inums[i]complement chahiyemap mein hai?Action
027map khaali haiNahi mila → map mein store karo {2:0}
172{2:0}Mil gaya! 2 index 0 pe tha → answer [0,1]
Map<Integer,Integer> seen = new HashMap<>(); for (int i = 0; i < nums.length; i++) { int complement = target - nums[i]; if (seen.containsKey(complement)) { return new int[]{seen.get(complement), i}; // mil gaya! } seen.put(nums[i], i); // check karne ke BAAD store karo }
⚠️ Sabse common mistakeCurrent number ko check karne se pehle map mein daal dena. Isse galat self-match ho sakta hai — jaise target=4 ho aur tum 2 pe ho, agar 2 ko pehle hi store kar diya, toh "2 ka complement 2 hai" wala check apne hi aapko match kar dega, jo galat hai. Hamesha pehle check karo, phir store karo.

5 Kadane's Algorithm
Yeh kis problem ko solve karta hai
Kadane's Algorithm tumhe array ke andar woh contiguous subarray dhundne mein help karta hai jiska sum sabse zyada ho — sirf O(n) time mein, bina har possible subarray check kiye.

Idea bahut simple hai: array mein chalte waqt, har position pe ek decision lo — "current number ko pichle chal rahe subarray mein jodu, ya yahan se ek naya subarray shuru karu?" Agar pichla sum negative ho gaya hai, toh usse aage carry karne ka koi fayda nahi — naya shuru karo.

Example: [-2, 1, -3, 4, -1, 2, 1, -5, 4]
iarr[i]curSummaxSumKya socha
0-2-2-2Shuru
1111-2+1=-1 vs naya start 1 → 1 behtar hai, restart
2-3-211-3=-2 vs naya start -3 → -2 behtar
3444-2+4=2 vs naya start 4 → 4 behtar, restart
4-1344-1=3, isse aage continue karna sahi hai
52553+2=5, continue
61665+1=6, naya max!

Final answer = 6, subarray [4, -1, 2, 1] se aaya. Dekho kaise jab bhi sum negative ho jaata, hum usse chhod ke naya start karte — kyunki negative sum ko aage carry karna sirf future sum ko kam karega.

int maxSum = arr[0], curSum = arr[0]; for (int i = 1; i < arr.length; i++) { curSum = Math.max(arr[i], curSum + arr[i]); // continue karu ya restart? maxSum = Math.max(maxSum, curSum); }

6 Prefix Sum
Simple shabdon mein
Prefix Sum ek naya array banata hai jisme prefix[i] = array ke shuru se index i tak ke sab elements ka total sum. Ek baar yeh ban jaaye, toh kisi bhi range (i, j) ka sum O(1) mein nikal sakte ho — har baar dobara add karne ki zarurat nahi.

Socho tum baar-baar poocha jaa rahe ho "index 2 se 5 tak ka sum kya hai?", "index 0 se 3 tak ka sum kya hai?" — agar har baar manually add karoge, slow hoga. Prefix sum array bana lo, fir formula hai: sum(i,j) = prefix[j] - prefix[i-1].

Isko HashMap ke saath combine karke ek bada useful trick milta hai — "kitne subarrays ka sum exactly K hai?" jaisa sawaal O(n) mein solve ho jaata hai. Trick yeh hai: jab bhi (currentSum - K) pehle kabhi dekha gaya ho, matlab beech ka koi subarray hai jiska sum exactly K hai.

7 Binary Search on Arrays
Binary Search kya hota hai
Socho ek dictionary mein koi word dhundna hai. Tum poora dictionary shuru se nahi padhte — beech mein khol ke dekhte ho, decide karte ho word usse pehle hai ya baad mein, aur sirf usi half ko aage check karte ho. Yehi Binary Search hai.

Yeh sirf sorted array pe kaam karta hai. Har step pe tum array ke beech wale element (mid) ko check karte ho — agar target usse chhota hai, toh poora right half discard kar do (kyunki sorted hai, target wahan nahi ho sakta). Agar bada hai, left half discard karo.

Har step pe search space aadha ho jaata hai — isliye yeh sirf O(log n) leta hai, jabki normal linear search O(n) leta. 10 lakh elements ke array mein bhi binary search sirf ~20 steps mein answer dhund leta hai.

Rotated sorted array (jaise [4,5,6,7,0,1,2]) mein thoda twist hai — poora array sorted nahi dikhta, lekin agar tum dhyan se dekho, ek half hamesha sorted hota hai. Pehle check karo kaunsa half (left ya right) sorted hai, fir decide karo target us sorted half ki range mein hai ya nahi.

8 Sorting-based Tricks

Kabhi-kabhi array ko pehle sort kar dena problem ko bahut aasan bana deta hai. Jaise 3Sum mein agar array sort kar do, toh Two Pointer use kar sakte ho. Merge Intervals mein agar intervals ko start time se sort kar do, toh overlapping intervals automatically ek doosre ke paas aa jaate hain, unhe dhundna easy ho jaata hai.

💡 Trade-off yaad rakhoSorting O(n log n) time leta hai. Agar tumhe O(n) solution chahiye (jaise Kadane's Algorithm), sorting use mat karo — woh tumhara time complexity worsen kar dega.

9 Common Mistakes

Yeh woh galtiyan hain jo sabse zyada hoti hain, chahe concept samajh mein aa gaya ho. Inhe yaad rakho:

1. Off-by-one errors — while (l < r) aur while (l <= r) mein farak hota hai. Hamesha socho: kya l aur r same index pe mil sakte hain ya nahi, aur usi ke hisaab se condition likho.

2. Empty array ka case bhool jana — agar array khaali ho ya sirf 1 element ho, tumhara code crash to nahi karega? Hamesha edge cases pehle soch lo.

3. Integer overflow — bahut bade numbers add karte waqt int ki limit cross ho sakti hai. Agar values bahut badi ho sakti hain, long use karo.

4. HashMap mein check-then-insert ka order — jaisa Two Sum mein dekha, pehle check karo "kya complement hai", phir current number ko store karo. Order ulta karne se galat answer aata hai.

5. Sliding Window mein if vs while — shrink karte waqt while use karo, if nahi, taaki jab zarurat ho tab multiple baar shrink ho sake.
