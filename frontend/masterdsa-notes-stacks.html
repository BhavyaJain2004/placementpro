1 Stack — Kya Hota Hai?
Real life analogy
Stack bilkul plate ki stack jaisi hoti hai — plates ek ke upar ek rakhte ho, aur hamesha sabse upar wali plate pehle uthate ho. Neeche wali plate tab tak nahi milti jab tak upar ki saari plates uthao. Yahi Stack ka rule hai: Last In, First Out (LIFO).

Stack mein sirf do main operations hote hain:

Push — element daalo (top pe). Pop — element nikalo (top se). Dono O(1) time mein hote hain — instant, size se farak nahi padta.

Push karne ke baad:

20
10
BOTTOM

Pop karne ke baad (30 nikla):

← TOP: 20

Stack ka use kab hota hai? Jab tumhe "pichli cheez yaad rakhni ho" — jaise browser ka back button (pichle pages yaad rakhta hai), undo/redo feature, expression evaluation, ya nested structures (brackets, HTML tags) check karna.

2 Stack Implement Karna
Java mein built-in Stack use karna
import java.util.*; Stack<Integer> st = new Stack<>(); st.push(10); // [10] st.push(20); // [10, 20] st.push(30); // [10, 20, 30] int top = st.peek(); // 30 (sirf dekha, nikala nahi) int popped = st.pop();// 30 nikal gaya → [10, 20] boolean empty = st.isEmpty(); // false
💡 ImportantJava mein Stack class legacy hai — interviews mein Deque<Integer> st = new ArrayDeque<>() prefer karo. Deque faster hai aur thread-safe issues nahi hote. push ki jagah offerLast(), pop ki jagah pollLast(), peek ki jagah peekLast() use karo.
Array se Stack banana (conceptually samajhne ke liye)
class MyStack { int[] arr; int top = -1; MyStack(int size) { arr = new int[size]; } void push(int val) { if (top == arr.length - 1) throw new RuntimeException("Stack full!"); arr[++top] = val; // top badhao, phir value daalo } int pop() { if (top == -1) throw new RuntimeException("Stack empty!"); return arr[top--]; // value lo, phir top ghata do } int peek() { if (top == -1) throw new RuntimeException("Stack empty!"); return arr[top]; } boolean isEmpty() { return top == -1; } }

Ye samajhna zaroori hai kyunki interviews mein kabhi bhi poocha ja sakta hai "stack without built-in implement karo". Array se karna O(1) time, O(n) space leta hai.

3 Valid Parentheses — Stack ka Classic Use

Ye problem stack ka textbook example hai aur almost har interview mein ek baar zaroor aata hai (ya iska variant). Problem: given a string like "()[]{}" ya "([)]" — kya ye valid hai?

Rule: Har opening bracket ((, [, {) ko uska matching closing bracket (), ], }) milna chahiye, sahi order mein.

Step-by-step trace: "({[]})"

]Closing ] — pop karo, top [ hai — match ✓[ (, { ]
}Closing } — pop karo, top { hai — match ✓[ ( ]

public boolean isValid(String s) { Deque<Character> st = new ArrayDeque<>(); for (char c : s.toCharArray()) { if (c == '(' || c == '[' || c == '{') { st.push(c); // opening bracket — push karo } else { if (st.isEmpty()) return false; // closing bracket bina opening ke char top = st.pop(); if (c == ')' && top != '(') return false; if (c == ']' && top != '[') return false; if (c == '}' && top != '{') return false; } } return st.isEmpty(); // sab match hua toh empty hogi }
⚠️ Edge case mat bhooloReturn karte waqt sirf true mat likho — st.isEmpty() likho. Warna "((" (sirf opening brackets) ke case mein galat true return ho jaayega kyunki loop mein koi closing bracket nahi mila jo fail karta.

4 Monotonic Stack — Sabse Important Pattern
Monotonic Stack kya hai
Monotonic stack ek normal stack hai jisme ek extra rule hota hai — stack ka har element ya toh hamesha increasing ya hamesha decreasing order mein hota hai (bottom se top tak). Jab bhi naya element aata hai jo is order ko tod raha ho, hum purane elements pop kar dete hain.

Ye pattern tab use hota hai jab tumhe har element ke liye "next greater element" ya "previous smaller element" dhundna ho — O(n²) brute force ko O(n) mein convert kar deta hai.

Example — Daily Temperatures

Array: [73, 74, 75, 71, 69, 72, 76, 73]. Har index ke liye answer chahiye — "kitne din baad temperature zyada milega?" Output: [1, 1, 4, 2, 1, 1, 0, 0].

Idea: stack mein indices store karo (values nahi). Jab current element stack ke top (index) wale element se zyada ho, pop karo aur difference calculate karo.

itemp[i]Stack (indices)Action
073[0]Stack khaali tha, push 0
174[1]74 > temp[0]=73 → pop 0, ans[0]=1-0=1, push 1
275[2]75 > temp[1]=74 → pop 1, ans[1]=2-1=1, push 2
371[2,3]71 < temp[2]=75 → push 3
469[2,3,4]69 < 71 → push 4
572[2,3,5]72 > 69 → pop 4, ans[4]=5-4=1. 72 > 71 → pop 3, ans[3]=5-3=2. 72 < 75 → push 5

public int[] dailyTemperatures(int[] temps) { int[] ans = new int[temps.length]; Deque<Integer> st = new ArrayDeque<>(); // indices store karo for (int i = 0; i < temps.length; i++) { // jab tak stack non-empty aur current temp > stack top ka temp while (!st.isEmpty() && temps[i] > temps[st.peek()]) { int idx = st.pop(); ans[idx] = i - idx; // kitne din baad mila } st.push(i); // index push karo } return ans; // remaining indices ka ans already 0 hai (default) }
💡 Key insightMonotonic stack problems mein values ki jagah indices store karo stack mein — taaki "kitne door" calculate kar sako. Sirf value store karoge toh distance nahi nikal payoge.
Min Stack — O(1) mein minimum element

Ye ek design problem hai: ek stack banao jo normal push/pop/top ke saath getMin() bhi O(1) mein de. Trick: ek auxiliary stack banao jo sirf minimums track kare.

class MinStack { Deque<Integer> st = new ArrayDeque<>(); Deque<Integer> minSt = new ArrayDeque<>(); // parallel min-tracking stack public void push(int val) { st.push(val); // minSt mein current minimum push karo (purana min ya naya val, jo chhota ho) int curMin = minSt.isEmpty() ? val : Math.min(val, minSt.peek()); minSt.push(curMin); } public void pop() { st.pop(); minSt.pop(); // dono stacks sync mein hain, saath pop karo } public int top() { return st.peek(); } public int getMin() { return minSt.peek(); } // O(1)! }

6 Queue Using Two Stacks

Ye ek classic design interview question hai — sirf do stacks use karke ek queue banao jisme enqueue (push) aur dequeue (pop) dono O(1) amortized time mein ho.

Idea: Do stacks rakho — inStack (naye elements aate hain) aur outStack (purane elements nikalte hain). Jab outStack khaali ho, poora inStack outStack mein transfer karo — isse order reverse ho jaata hai (LIFO → FIFO).

OperationinStackoutStackAction
push(1)[1][]inStack mein daalo
push(2)[1,2][]inStack mein daalo
push(3)[1,2,3][]inStack mein daalo
pop()[][3,2,1]outStack khaali tha → sab transfer karo, phir pop → 1 nikla
pop()[][3,2]outStack mein 2 hai → seedha pop → 2 nikla
push(4)[4][3,2]sirf inStack mein daalo, outStack disturb mat karo
pop()[4][3]outStack mein abhi 3 hai → pop → 3 nikla
class MyQueue { Deque<Integer> inStack = new ArrayDeque<>(); Deque<Integer> outStack = new ArrayDeque<>(); public void push(int x) { inStack.push(x); // hamesha inStack mein daalo } public int pop() { peek(); // outStack khaali ho toh transfer karo return outStack.pop(); } public int peek() { if (outStack.isEmpty()) { while (!inStack.isEmpty()) { outStack.push(inStack.pop()); // poora transfer — order reverse hoga } } return outStack.peek(); } public boolean empty() { return inStack.isEmpty() && outStack.isEmpty(); } }
💡 Amortized O(1) kyun?Har element sirf ek baar inStack se outStack mein transfer hota hai — total O(n) transfers n elements ke liye, isliye per-operation average O(1) hai. Transfer hamesha tab hi hota hai jab outStack khaali ho.

7 Deque & Sliding Window Maximum
Deque kya hota hai
Double-Ended Queue (Deque) — isme dono taraf se elements add aur remove kar sakte ho. Front se bhi, back se bhi. Ye stack + queue dono ka kaam karta hai. Java mein ArrayDeque hi deque hai.

Deque ka sabse famous interview use case hai — Sliding Window Maximum. Tumhe ek array diya gaya hai aur ek window size k — har window ke andar ka maximum element return karo.

Brute force: har window ke liye max dhundho = O(n×k). Deque approach: O(n) mein sab windows ka max nikal sakte ho.

Approach — Monotonic Decreasing Deque

Deque mein indices store karo, aur condition maintain karo: deque mein stored indices ke values hamesha decreasing order mein hon (front mein sabse bada). Jab naya element aata hai, peeche se chhote elements pop karo (woh kabhi maximum ban nahi sakte).

inums[i]Deque (indices)Window Max
01[0]—
13[1]—(3 > 1, so pop 0, push 1)
2-1[1,2]3 (window [1,3,-1], front=1, nums[1]=3)
3-3[1,2,3]3 (window [3,-1,-3], front=1 still valid)
45[4]5 (5 > all, pop 3,2,1; front=4, nums[4]=5)
53[4,5]5 (window [5,3,...], front=4)
66[6]6 (6 > all)
77[7]7
public int[] maxSlidingWindow(int[] nums, int k) { Deque<Integer> dq = new ArrayDeque<>(); // indices store karega int[] result = new int[nums.length - k + 1]; for (int i = 0; i < nums.length; i++) { // front se: window ke bahar chale gaye indices remove karo if (!dq.isEmpty() && dq.peekFirst() < i - k + 1) { dq.pollFirst(); } // back se: current element se chhote indices remove karo (woh kabhi max nahi honge) while (!dq.isEmpty() && nums[dq.peekLast()] < nums[i]) { dq.pollLast(); } dq.offerLast(i); // current index add karo // window complete hone ke baad result store karo if (i >= k - 1) { result[i - k + 1] = nums[dq.peekFirst()]; // front = maximum } } return result; }
⚠️ Yaad rakhoWindow se bahar gaye index ko first remove karo (front se), phir chhote elements remove karo (back se). Order matter karta hai — pehle expiry check, phir monotonic property maintain karo.

8 Common Mistakes & Interview Tips

1. isEmpty() check bhoolo mat — Stack/Queue pe pop ya peek karne se pehle hamesha isEmpty() check karo, warna runtime exception aayega. Ye sabse common bug hai.

2. Stack mein values nahi, indices store karo — Monotonic stack problems mein mostly tum distance ya position calculate karte ho, toh values ki jagah indices store karna zaroori hai.

3. Deque vs Stack vs Queue — Java mein confusion — Java mein ArrayDeque sabse versatile hai, stack aur queue dono ki jagah use karo. Stack ke liye push/pop/peek, Queue ke liye offer/poll/peek use karo same ArrayDeque pe.

4. Circular Queue ki boundary conditions — Agar circular queue implement karna ho, front aur rear pointers ko modulo se manage karo. Edge cases: full queue check (rear + 1) % size == front, empty check front == rear.

5. "Stack se kya solve hoga" pehchano — Agar koi nested structure hai (brackets, directories), ya "pichla element" yaad rakhna ho, ya "next greater/smaller" dhundna ho — Stack sochna shuru karo. Queue ki zarurat tab hoti hai jab level-by-level ya FIFO order chahiye.

Stack aur Queue simple concepts hain lekin unke creative use cases — monotonic stack, min stack, two-stack queue, sliding window deque — ye sab pattern-recognition ke sawaal hain. Jitna practice karoge utna pehle se pehchaan paoge "ye monotonic stack wala sawaal hai". Ab Practice page pe jao aur 20 questions solve karo.
