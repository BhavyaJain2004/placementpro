Recursion

Recursion ek aisa concept hai jise log pehli baar dekhte hain toh confusing lagta hai — "function khud ko kyun call kar raha hai?" — lekin jab ek baar samajh aa jaata hai toh problems ek alag nazar se dikhne lagte hain. Recursion sirf ek tool nahi hai, ek sochne ka tarika hai. Trees, Graphs, Backtracking, Dynamic Programming — sab ki neenv recursion mein hai. Chalo bilkul zero se shuru karte hain.

1 Recursion Kya Hai?
Simple definition
Recursion ek technique hai jisme ek function khud ko call karta hai apni problem ka ek chhota version solve karne ke liye — tab tak jab tak problem itni chhoti ho jaaye ki directly answer pata ho (base case).

Real-life analogy: Socho tumhare paas ek badi Russian doll hai. Usme ek chhoti doll hai, usme aur chhoti, aur aage bhi. Tumhe sirf ek kaam karna hai — kholo aur andar dekho. Jab tak koi aur doll na ho (base case), bas wahi return karo. Har doll apne andar wali doll ka kaam delegate karti hai — recursion bilkul aise hi kaam karta hai.

Call Stack kaise kaam karta hai — Factorial example

factorial(4) = 4 × 3 × 2 × 1 = 24. Recursively: factorial(n) = n × factorial(n-1), base case: factorial(0) = 1.

int factorial(int n) { if (n == 0) return 1; // base case return n * factorial(n-1); // recursive case }

Jab tum factorial(4) call karte ho, computer ke andar kya hota hai:

factorial(4) calls → factorial(3) factorial(3) calls → factorial(2) factorial(2) calls → factorial(1) factorial(1) calls → factorial(0) factorial(0) = 1 ← BASE CASE, wapas aana shuru factorial(1) = 1 × 1 = 1 factorial(2) = 2 × 1 = 2 factorial(3) = 3 × 2 = 6 factorial(4) = 4 × 6 = 24 ← Final Answer

Ye call stack hai — computer har function call ko ek "frame" mein save karta hai. Jab tak function return nahi karta, woh frame stack pe bana rehta hai. Isliye recursion memory use karta hai O(depth) amount.

2 Base Case — Recursion Ka Lifeline

Base case woh condition hai jahan recursion ruk jaati hai aur seedha answer return karta hai bina khud ko call kiye. Base case ke bina recursion infinite loop mein chali jaati hai aur program crash ho jaata hai (Stack Overflow Error).

Base case dhundhne ka formula

Ye 3 sawaal khud se poochho:
1. Problem ka sabse chhota version kya hai jiska answer obvious hai?
2. Kab recursion ko rukna chahiye?
3. Kya main har call mein problem genuinely chhoti kar raha hoon?

ProblemBase CaseReduction
Factorial(n)n == 0 → return 1n → n-1
Fibonacci(n)n ≤ 1 → return nn → n-1, n-2
Reverse stringlength ≤ 1 → return ss → s[1:]
Binary searchlo > hi → return -1half array each call
Tree traversalnode == null → returnnode → children
Subsets generationindex == n → save resultindex → index+1
⚠️ Infinite Recursion se bachaoAgar base case galat likha ho, ya recursive call problem ko genuinely chhota nahi kar rahi — Stack Overflow aayega. Jaise factorial(-1) call karo toh n kabhi 0 nahi hoga — crash! Isliye inputs validate karo ya base case mein n <= 0 use karo.
3 Recursive Sochna Kaise Seekhein
The Golden Rule of Recursion
"Trust the recursion." Jab tum recursive function likhte ho, ye mat socho ki exactly kaise kaam karega step-by-step. Bas trust karo ki func(smallerProblem) sahi answer dega — tumhara kaam sirf current step handle karna hai.

Example: String reverse karo recursively.

Sochne ka tarika: "Agar main reverse("ello") pe trust karta hoon ki woh "olle" dega, toh reverse("hello") = reverse("ello") + "h" = "olleh""

String reverse(String s) { if (s.length() <= 1) return s; // base case return reverse(s.substring(1)) + s.charAt(0); // trust recursion for rest }
CallReturns
reverse("hello")reverse("ello") + 'h'
reverse("ello")reverse("llo") + 'e'
reverse("llo")reverse("lo") + 'l'
reverse("lo")reverse("o") + 'l'
reverse("o")"o" (base case)
Unwinding..."o"+"l"+"l"+"e"+"h" = "olleh" ✅
3-Step Template for Any Recursive Problem
// Step 1: Base case (when to stop) if (baseCondition) return baseAnswer; // Step 2: Recursive call (smaller subproblem) ResultType subResult = solve(smallerInput); // Step 3: Combine (use subResult to build current answer) return combine(currentElement, subResult);

4 Tree Recursion — Fibonacci Example

Ab tak humne "linear recursion" dekhi — ek call, ek subproblem. Tree recursion mein ek function multiple recursive calls karta hai — tree ki tarah branching hoti hai.

Fibonacci — Naive Recursion

Fibonacci sequence: 0, 1, 1, 2, 3, 5, 8, 13... Rule: fib(n) = fib(n-1) + fib(n-2), base case: fib(0)=0, fib(1)=1.

int fib(int n) { if (n <= 1) return n; // base case return fib(n-1) + fib(n-2); // TWO recursive calls — tree recursion }

fib(5) ka call tree kaisa dikhta hai:

fib(5) / \ fib(4) fib(3) / \ / \ fib(3) fib(2) fib(2) fib(1) / \ / \ / \ fib(2) fib(1) fib(1) fib(0) fib(1) fib(0) / \ fib(1) fib(0) Total calls for fib(5) = 15 calls! For fib(50) = 2^50 calls ≈ 10^15 😱 → Too slow!

Notice: fib(3) do baar calculate ho raha hai, fib(2) teen baar! Ye overlapping subproblems hain — aur iska solution hai Memoization (Dynamic Programming topic mein aayega).

Memoization se fix karo — O(2^n) → O(n)
int[] memo = new int[100]; Arrays.fill(memo, -1); int fib(int n) { if (n <= 1) return n; if (memo[n] != -1) return memo[n]; // pehle se calculate? seedha return karo memo[n] = fib(n-1) + fib(n-2); // calculate karo aur save karo return memo[n]; }
💡 Key insightJab bhi recursive solution mein same subproblem baar-baar solve ho raha ho — memoization use karo. Ek simple array ya HashMap se O(2^n) → O(n) ho jaata hai. Ye DP ka pehla step hai.

5 Backtracking — Try, Explore, Undo
Backtracking kya hai
Backtracking ek technique hai jisme tum ek decision lete ho, aage explore karte ho, aur agar galat rasta mila toh us decision ko undo karke dusra option try karte ho. "Try every possibility, but backtrack when stuck."

Socho tum ek maze mein ho. Ek direction try karo, agar dead end mila — wapas ao aur doosra raasta try karo. Backtracking exactly yehi hai, bas code mein.

Template — Backtracking ka universal structure
void backtrack(State current, List<State> result) { // Step 1: Base case — kya hum destination pe pahunche? if (isComplete(current)) { result.add(new State(current)); // valid solution save karo return; } // Step 2: Har possible choice try karo for (Choice c : getPossibleChoices(current)) { if (isValid(c)) { // ye choice valid hai? applyChoice(current, c); // choice lo (state update karo) backtrack(current, result); // aage explore karo undoChoice(current, c); // UNDO — wapas aao (backtrack!) } } }
Example — Subsets Generate Karo

Input: [1,2,3]. Har element ke liye 2 choices hain — include karo ya skip karo. Total 2^3 = 8 subsets.

backtrack(idx=0, current=[]) ├── Include 1: backtrack(idx=1, current=[1]) │ ├── Include 2: backtrack(idx=2, current=[1,2]) │ │ ├── Include 3: Save [1,2,3] │ │ └── Skip 3: Save [1,2] │ └── Skip 2: backtrack(idx=2, current=[1]) │ ├── Include 3: Save [1,3] │ └── Skip 3: Save [1] └── Skip 1: backtrack(idx=1, current=[]) ├── Include 2: backtrack(idx=2, current=[2]) │ ├── Include 3: Save [2,3] │ └── Skip 3: Save [2] └── Skip 2: backtrack(idx=2, current=[]) ├── Include 3: Save [3] └── Skip 3: Save []
void backtrack(int[] nums, int idx, List<Integer> current, List<List<Integer>> result) { if (idx == nums.length) { result.add(new ArrayList<>(current)); // copy save karo! return; } // Choice 1: include nums[idx] current.add(nums[idx]); backtrack(nums, idx + 1, current, result); current.remove(current.size() - 1); // UNDO — remove last element // Choice 2: skip nums[idx] backtrack(nums, idx + 1, current, result); }
⚠️ Sabse important — Copy save karo!Jab base case mein current state save karte ho, result.add(current) mat karo — ye reference save hoga, aur backtracking ke baad list change ho jaayegi. Hamesha result.add(new ArrayList<>(current)) karo — ek naya copy banao.
Permutations — Used[] array pattern

Subsets mein order matter nahi karta, permutations mein karta hai. Har position pe har unused element try karo:

void backtrack(int[] nums, boolean[] used, List<Integer> current, List<List<Integer>> result) { if (current.size() == nums.length) { result.add(new ArrayList<>(current)); return; } for (int i = 0; i < nums.length; i++) { if (used[i]) continue; // pehle se use hua hai, skip used[i] = true; // mark as used current.add(nums[i]); backtrack(nums, used, current, result); current.remove(current.size() - 1); // UNDO used[i] = false; // UNDO mark } }
N-Queens — Pruning ke saath backtracking

N-Queens mein N×N board pe N queens rakhni hain taaki koi ek doosre ko attack na kare. Backtracking yahan powerful hai kyunki hum invalid placements ko pehle hi discard kar sakte hain (pruning) — poori branch explore kiye bina.

Conflict check: same column (cols set), same diagonal (row+col), same anti-diagonal (row-col) — teen sets maintain karo, O(1) mein check hoga.

Set<Integer> cols = new HashSet<>(); Set<Integer> diag1 = new HashSet<>(); // row+col same = same diagonal Set<Integer> diag2 = new HashSet<>(); // row-col same = same anti-diagonal int solve(int row, int n) { if (row == n) return 1; // sab queens placed successfully int count = 0; for (int col = 0; col < n; col++) { if (cols.contains(col) || diag1.contains(row+col) || diag2.contains(row-col)) continue; // conflict hai, skip this column cols.add(col); diag1.add(row+col); diag2.add(row-col); // place queen count += solve(row + 1, n); cols.remove(col); diag1.remove(row+col); diag2.remove(row-col); // UNDO } return count; }

6 Divide and Conquer
Pattern
Problem ko independent subproblems mein divide karo, har ek recursively solve karo, phir results combine karo. Backtracking se alag hai — yahan subproblems independent hote hain (ek doosre ko affect nahi karte).

Classic examples: Merge Sort, Quick Sort, Binary Search, Fast Power.

Merge Sort — Divide and Conquer ka perfect example
void mergeSort(int[] arr, int left, int right) { if (left >= right) return; // base case: 1 ya 0 elements already sorted int mid = (left + right) / 2; mergeSort(arr, left, mid); // left half sort karo mergeSort(arr, mid + 1, right); // right half sort karo merge(arr, left, mid, right); // combine (merge) karo }
PhaseArray state
Original[5, 2, 4, 1, 3]
Divide until single[5] [2] [4] [1] [3]
Merge pairs[2,5] [1,4] [3]
Merge again[1,2,4,5] [3]
Final merge[1,2,3,4,5] ✅
Fast Power — O(n) → O(log n)

x^n ko n baar multiply karne ki jagah: x^n = (x^(n/2))^2 agar n even, ya x × (x^((n-1)/2))^2 agar n odd. Har step mein n half hota hai — O(log n) calls!

double fastPow(double x, long n) { if (n == 0) return 1.0; // base case: x^0 = 1 double half = fastPow(x, n / 2); // n/2 ke liye recursion (divide) if (n % 2 == 0) return half * half; // n even: half^2 else return half * half * x; // n odd: half^2 × x }

7 Tail Recursion & Stack Overflow

Stack Overflow tab hota hai jab recursion bahut deep ho jaaye aur call stack ka memory khatam ho jaye. Typically Java mein ~5000-10000 calls ke baad crash ho sakta hai (depends on stack size).

Tail Recursion ek optimization hai — jab recursive call function ka last operation ho (return ke baad koi computation na ho), toh compiler/JVM us call ko reuse kar sakta hai instead of naya stack frame banana. Java mein ye automatically nahi hota (unlike languages jaise Scala/Haskell), lekin concept samajhna zaroori hai.

// NON-tail recursive — return ke baad multiplication hoti hai int factorial(int n) { if (n == 0) return 1; return n * factorial(n-1); // n * (...) — extra work after return } // Tail recursive — accumulator pattern int factTail(int n, int acc) { if (n == 0) return acc; // accumulator hi final answer hai return factTail(n-1, n * acc); // koi extra work nahi return ke baad } // Call: factTail(5, 1)
💡 Practical tipAgar recursion bahut deep hogi (jaise n = 100000), iterative solution prefer karo interviews mein — explicitly Stack/Queue use karke. Recursion ko iterative mein convert karna bhi ek important skill hai.
8 Common Mistakes & Must-Do Questions

1. Base case miss karna ya galat likhna — Pehle base case socho, baad mein recursive case. Test edge cases manually: n=0, empty array, single element.

2. List copy na karna backtracking mein — result.add(current) ki jagah result.add(new ArrayList<>(current)) — nahi kiya toh sab results khali list ho jaate hain end mein.

3. Undo (backtrack) step bhoolo mat — Har applyChoice ke baad undoChoice hona chahiye, warna state corrupt ho jaati hai agli calls ke liye.

4. Recursion mein global variables use karna — Global state ke saath recursion bahut buggy hoti hai. Function arguments ya return values prefer karo.

5. "Start index" pass karna bhool jaana combination problems mein — Combination Sum jaisi problems mein start index pass karo taaki duplicates na aayein aur peeche na jaao.
